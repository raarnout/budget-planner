# AI Agent Guide: Next.js, Prisma, CSS Modules, Atomic Design, Storybook & 100% Coverage

You are an expert AI software engineer. Follow these rules, architecture patterns, and conventions strictly for all code modifications.

## 1. Project Context

- **Framework**: Next.js (App Router with local documentation reference)
- **Language**: TypeScript (Strict mode)
- **Database ORM**: Prisma ORM
- **Database**: PostgreSQL
- **Styling**: CSS Modules (Native Next.js styling)
- **Architecture**: Atomic Design Pattern
- **Component Documentation**: Storybook
- **Quality Assurance**: Unit Testing (Vitest / Jest + React Testing Library) with strict **100% Code Coverage** requirement.

## 2. Directory Structure (Atomic Design, Storybook & Tests)

All UI components must be structured inside the `/components` directory. Component files, style modules, stories, and tests must all be colocated in the exact same directory:

- **`components/atoms/Button/`**
  - `Button.tsx` (Component code)
  - `Button.module.css` (Component scoped styles)
  - `Button.stories.tsx` (Storybook documentation)
  - `Button.test.tsx` (Unit tests achieving 100% coverage)

## 3. Layout & Encapsulation Rules (Strict)

- **No External Spacing**: Components (Atoms, Molecules, Organisms) must **never** have outer `margin` or outer `padding` on their root/outermost HTML tag.
- **Exact Sizing**: The root tag of a component must represent the exact bounding box/dimensions of the component itself.
- **Parent Responsibility**: Positioning and spacing _between_ components are strictly controlled by the parent component using layout systems like CSS Flexbox (`gap`) or CSS Grid.
- **Molecule Rule**: When positioning multiple Atoms inside a Molecule, the Molecule's CSS determines their layout and spacing. However, the Molecule's own root tag must still have zero outer margin or padding.

## 4. Testing & 100% Code Coverage Rules (Strict)

- **Mandatory 100% Coverage**: Every newly created or modified component must have a `.test.tsx` file that achieves **100% line, branch, function, and statement coverage**.
- **Test All Branches**: You must explicitly test all conditional rendering logic (`&&`, ternaries `? :`, switch cases, and optional chaining).
- **Test User Interaction**: Fire events (clicks, type, focus) using `@testing-library/user-event` to test interactive behaviors and assertions.
- **Mocking Extraneous Layer**: For Server Actions or complex data layers, mock any Prisma client calls to ensure testing states are fully controlled.
- **Error Boundaries**: Always test the failure paths (e.g., how the component responds if data fetching fails or if a function throws an error).

## 5. Storybook Rules

- **CSF3 Format**: Every Atom, Molecule, and Organism component must have a corresponding `.stories.tsx` file using object-based exports (`Meta` and `StoryObj`).
- **No Database Operations**: Provide realistic mock data in the stories. Do not run Prisma operations inside Storybook files.

## 6. Next.js App Router & Prisma Rules

- Use Server Components by default. Keep data fetching inside the `app/` page layer using `async/await`.
- **Prisma Client Instance**: Always import the single, global Prisma instance from `@/lib/db`. Never instantiate `new PrismaClient()` directly to avoid connection leaks.
- **Type Safety**: Use types generated by Prisma (e.g., `import { User } from '@prisma/client'`).

## 7. Styling Rules (CSS Modules)

- **File Naming**: Name styling files using the `.module.css` extension.
- **Naming Conventions**: Use camelCase for CSS class names (e.g., `.mainContainer` instead of `.main-container`) to allow clean JavaScript dot-notation imports.
- **Import Pattern**: Always import styles as an object: `import styles from './Button.module.css';`

### [ATOM] Component: `components/atoms/Button/Button.tsx`

```tsx
import styles from './Button.module.css';

export interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: 'primary' | 'secondary';
  isLoading?: boolean;
}

export function Button({
  children,
  variant = 'primary',
  isLoading = false,
  ...props
}: ButtonProps) {
  return (
    <button
      className={`${styles.btn} ${styles[variant]}`}
      disabled={isLoading || props.disabled}
      {...props}
    >
      {isLoading ? (
        <span data-testid="loading-spinner">Loading...</span>
      ) : (
        children
      )}
    </button>
  );
}
```

### [ATOM] Test File (Achieving 100% Code Coverage): `components/atoms/Button/Button.test.tsx`

```tsx
import { render, screen } from '@testing-library/react';
import userEvent from '@testing-library/user-event';
import { describe, it, expect, vi } from 'vitest';
import { Button } from './Button';

describe('Button Component', () => {
  it('renders primary variant by default with children text', () => {
    render(<Button>Click me</Button>);
    const button = screen.getByRole('button', { name: /click me/i });
    expect(button).toBeInTheDocument();
    expect(button).not.toBeDisabled();
  });

  it('renders the secondary variant class correctly', () => {
    render(<Button variant="secondary">Secondary</Button>);
    const button = screen.getByRole('button', { name: /secondary/i });
    expect(button).toBeInTheDocument();
  });

  it('renders loading state, text is hidden, and button is disabled', () => {
    render(<Button isLoading={true}>Click me</Button>);
    const button = screen.getByRole('button');
    expect(button).toBeDisabled();
    expect(screen.getByTestId('loading-spinner')).toBeInTheDocument();
    expect(screen.queryByText('Click me')).not.toBeInTheDocument();
  });

  it('supports standard HTML disabled attribute', () => {
    render(<Button disabled>Disabled</Button>);
    const button = screen.getByRole('button', { name: /disabled/i });
    expect(button).toBeDisabled();
  });

  it('triggers onClick handler when clicked by user', async () => {
    const handleClick = vi.fn();
    render(<Button onClick={handleClick}>Trigger</Button>);
    const button = screen.getByRole('button', { name: /trigger/i });

    await userEvent.click(button);
    expect(handleClick).toHaveBeenCalledTimes(1);
  });
});
```
